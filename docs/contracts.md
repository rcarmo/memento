# Memento contracts

## Concept schema v1

Each concept is one UTF-8 Markdown file with YAML frontmatter.

Required fields:

- `schema_version`: integer `1`
- `id`: immutable UUID string generated by the service
- `type`: one of `concept`, `instance`, `person`, `project`, `service`, `system`
- `title`: non-empty string
- `status`: `active`, `deprecated`, or `tombstone`
- `created_at`: UTC RFC3339 timestamp
- `updated_at`: UTC RFC3339 timestamp
- `updated_by`: authenticated principal identifier

Optional fields:

- `description`
- `aliases`
- `tags`
- `source_refs`
- `supersedes`

Unknown top-level fields are rejected.

## MCP envelopes

Success envelope:

```json
{
  "status": "success",
  "data": {},
  "warnings": [],
  "next_tools": [],
  "repo_revision": "<git-sha>",
  "index_revision": "<git-sha>",
  "index_stale": false,
  "operation_id": null
}
```

Error envelope:

```json
{
  "status": "error",
  "error_class": "validation_error",
  "message": "human-readable summary",
  "warnings": [],
  "repo_revision": null,
  "index_revision": null,
  "index_stale": false,
  "operation_id": null
}
```

Stable error classes for the deterministic core:

- `validation_error`
- `not_found`
- `forbidden`
- `conflict`
- `idempotency_conflict`
- `repo_unavailable`
- `repo_dirty`
- `index_unavailable`
- `queue_full`
- `temporarily_read_only`

## Pagination

List and search responses will use deterministic cursors derived from sorted result order and validated limits. Pagination is documented now and implemented in later milestones.
